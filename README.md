## The Problem

Yogurt Park is frozen yogurt shop near the UC Berkeley campus. Its daily flavor lineup from a catalog of 66 flavors across **4 rotating slots** (plus 2 permanently fixed: Ghirardelli Chocolate and Vanilla Classic). Poor daily selections cost real money: customers leave when preferred flavors are unavailable, prep costs are wasted on unpopular choices, and unsold inventory must be discarded.

**Core question:** Which 4 flavors should rotate each day to maximize customer utility?

---

## Approach

We built a **Multinomial Logit (MNL) choice model** using 308 days of Instagram engagement data (Jan–Nov 2025) as a proxy for customer preference, following Berry (1994)'s aggregate demand inversion framework.

### Data

Yogurt Park posts their daily flavor lineup on Instagram each morning. We scraped 308 posts, recording the 6 daily flavors and the number of likes each post received.

### Modeling pipeline

```
Raw Instagram data (308 days)
        ↓
  Data cleaning & feature engineering
  (9 flavor categories, descriptors, temporal features)
        ↓
  Poisson Regression (baseline)
  → confirms assortment composition matters
  → cannot model substitution or individual flavor utilities
        ↓
  Multinomial Logit — 94 features (failed)
  → quasi-complete separation (2.3 obs/parameter)
  → Pseudo R² = 1.94 (impossible)
        ↓
  MNL — 22 features + L2 regularization (final model)
  → 39.8 obs/parameter, converged, stable coefficients
        ↓
  Plackett-Luce simulation (validation)
  → 1,000 customers × 4 contexts × 4 strategies
```

## Key Findings

### Context-dependent preferences

| Interaction | Effect |
|---|---|
| Weekend × Sorbet | **+248%** |
| Summer × Chocolate | **+166%** |
| Summer × Tart | **+140%** |
| Weekday × Vanilla | **+90%** |
| Weekend × Nut | **+62%** |

These reversals are the core insight: **chocolate has −6% base utility but +166% in summer**. Sorbet has −12% base but +248% on weekends. Static "serve the most popular flavors" logic misses all of this.

### Category base effects (relative to Vanilla)

Coffee/Matcha (+92%) and Fruit (+62%) lead year-round. Mid-tier flavors (ranks 10–20) suffer a **−23% popularity penalty** — customers tire of seeing them without the draw of top-tier alternatives.

### Validation results

| Method | Mean Utility | Hit Rate | vs. Random |
|---|---|---|---|
| **MNL-Optimized** | **1.714** | **9.4%** | **+69%** |
| Current-Strategy | 1.059 | 6.7% | +4% |
| Random | 1.015 | 6.2% | — |
| Popularity-Only | 0.896 | 8.0% | −12% |

Statistical significance: MNL vs. Random (p=0.037), MNL vs. Current (p=0.042).

> **Most revealing finding:** MNL's largest gains occur in *non-obvious* contexts. Weekend × Fall shows +115% improvement over current strategy — exactly where human intuition is least reliable.

### Recommended rotation policy

| Context | Rotating Slot Priorities |
|---|---|
| Weekend (any season) | Sorbet, Nut, + seasonal fill |
| Weekend + Summer | Sorbet, Nut, Tart, Fruit |
| Weekday + Summer | Coffee/Matcha, Chocolate, Tart |
| Weekday + Spring | Coffee/Matcha, Nut, Fruit |
| Weekday + Fall/Winter | Coffee/Matcha, Fruit, Caramel/Dulce |

## Running the Notebooks

```bash
# Install dependencies
pip install -r requirements.txt

# Run in order
jupyter notebook notebooks/01_eda.ipynb
jupyter notebook notebooks/02_modeling.ipynb
jupyter notebook notebooks/03_validation.ipynb
```

Each notebook saves its outputs to `data/processed/` and is readable independently, but `02_modeling` depends on outputs from `01_eda`, and `03_validation` re-fits the final MNL model to remain self-contained.
